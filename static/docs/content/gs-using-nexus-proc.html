Managing Java artifacts using Nexus Repository Manager

The {PRODUCT} cluster includes a running instance of Nexus Repository Manager. If you are developing Java applications, you might want to use Nexus Repository Manager to cache and manage dependencies.

Using Nexus Repository Manager to build your applications

  1. Set the MAVEN_MIRROR_URL environment variable in your BuildConfig:

    MAVEN_MIRROR_URL=http://nexus.openshift-nexus.svc:8081/nexus/content/groups/public

    For non RHMI clusters, or custom installs, nexus may be installed in a different namespace. In those cases use the following:

    MAVEN_MIRROR_URL=http://nexus.nexus.svc:8081/nexus/content/groups/public

    This ensures that Maven uses Nexus Repository Manager.

  2. Build your application.

  3. Test your application.

Subsequent builds should be faster dues to the Nexus Repository Manager caching.
